# Security Policy

## Supported Surface

Security fixes should target `main`.

The project is still moving quickly, so only the latest mainline code is considered supported.

## Reporting

Use GitHub private vulnerability reporting if it is enabled for the repository.

If private reporting is not available, contact the maintainers directly before opening a public issue with exploit details.

## Response Goals

- Acknowledge valid reports within 3 business days.
- Confirm severity and next steps after triage.
- Publish a fix or mitigation note as soon as a safe patch is ready.

## Reporter Guidance

- Include affected routes, packages, and environment assumptions.
- Include reproduction steps and expected impact.
- Avoid public disclosure until maintainers confirm a fix or mitigation window.
